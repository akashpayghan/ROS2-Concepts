üîë Key Terms

  • Node: A small program that does one specific job (like reading sensors, moving motors)
  • Topic: A named channel where nodes send and receive messages
  • Message: The actual data being sent (like sensor readings, commands)

üéöÔ∏è Quality of Service (QoS) - Message Delivery Guarantee

QoS profiles control HOW messages are delivered. Think of it like shipping options:

Best Effort

  • ‚úÖ Fast, low latency
  • ‚ùå May lose messages
  • üìπ Use for: Camera streams, sensor data
  • üí° Like regular mail - fast but might get lost

Reliable

  • ‚úÖ Guaranteed delivery
  • ‚ùå Slightly slower
  • üéØ Use for: Commands, critical data
  • üí° Like certified mail - guaranteed delivery
# Setting QoS in Python from rclpy.qos import QoSProfile, ReliabilityPolicy, HistoryPolicy # For sensor data (fast, can lose some) sensor_qos = QoSProfile( reliability=ReliabilityPolicy.BEST_EFFORT, history=HistoryPolicy.KEEP_LAST, depth=10 ) # For critical commands (must arrive) command_qos = QoSProfile( reliability=ReliabilityPolicy.RELIABLE, history=HistoryPolicy.KEEP_ALL ) # Create publisher with QoS self.publisher = self.create_publisher( Image, '/camera/image', sensor_qos )

üéØ Best Practices for Production

  • Namespaces: Group related nodes (/drone1/camera, /drone1/controller)
  • Launch Files: Start multiple nodes together with parameters
  • Parameters: Configure nodes without recompiling (camera resolution, detection threshold)
  • Logging: Use ROS2 logging levels (DEBUG, INFO, WARN, ERROR)
  • TF2: Coordinate frame transformations (camera ‚Üí drone ‚Üí world)
  • Bags: Record and replay data for testing
# Advanced launch file example from launch import LaunchDescription from launch_ros.actions import Node def generate_launch_description(): return LaunchDescription([ # Camera node Node( package='camera_driver', executable='camera_node', name='siyi_camera', parameters=[{ 'resolution': '1920x1080', 'fps': 30 }] ), # Detector node Node( package='person_tracker', executable='yolo_detector', name='detector', parameters=[{ 'model_path': '/models/yolov8n.pt', 'confidence': 0.7 }], remappings=[ ('/image', '/camera/image_raw') ] ), # Gimbal controller Node( package='gimbal_control', executable='gimbal_node', name='gimbal' ) ])

üìä ROS2 vs ROS1: Why Upgrade?

‚ùå ROS1 Issues

  • Single point of failure (roscore)
  • No real-time support
  • TCP only (slower)
  • Python 2 based
  • No built-in security

‚úÖ ROS2 Advantages

  • Decentralized (no master)
  • Real-time capable (DDS)
  • UDP multicast (faster)
  • Python 3 & C++17
  • Built-in security (SROS2)
  • Better Windows support
