1.1.1 What is ROS2?

ROS2

ROS2 (Robot Operating System 2) is an open-source robotics middleware framework designed for building complex robot applications with real-time capabilities, improved security, and native support for distributed systems.

Core Concepts

üì°

Distributed Communication

ROS2 uses DDS (Data Distribution Service) for peer-to-peer communication, enabling multiple robots and computers to communicate seamlessly without a central master.

‚ö°

Real-Time Performance

Built with real-time systems in mind, ROS2 supports deterministic behavior crucial for drone flight control and sensor processing.

üîí

Security

Native support for DDS Security (SROS2) provides authentication, encryption, and access control for robotic systems.

üéØ

Quality of Service

Configurable QoS policies allow fine-tuning of reliability, durability, and latency for different communication needs.

üêç

Multi-Language Support

Native APIs for Python, C++, and community support for other languages like Rust and Java.

üîÑ

Lifecycle Management

Managed nodes with defined states (configuring, active, inactive) for better control over system behavior.

üí° For Drone Applications: ROS2's real-time capabilities are essential for controlling gimbal cameras, processing vision data from SIYI cameras, and maintaining stable communication with Pixhawk flight controllers.

Why ROS2 for Drones?

üéØ Low Latency

Critical for person tracking and gimbal control - ROS2 can achieve sub-millisecond latencies with proper QoS settings.

üåê Multi-Robot Support

Native support for drone swarms without complex networking setup. Perfect for coordinated missions.

‚öñÔ∏è Resource Efficiency

Optimized for embedded systems like Jetson Orin, with options for zero-copy and intra-process communication.

üîå Hardware Integration

Extensive support for cameras, IMUs, GPS, LiDAR through sensor_msgs and standard interfaces.

Code Comparison

ROS1 Publisher (Python)

import rospy from std_msgs.msg import String rospy.init_node('publisher') pub = rospy.Publisher( 'topic', String, queue_size=10 ) rate = rospy.Rate(10) while not rospy.is_shutdown(): pub.publish("Hello") rate.sleep()

ROS2 Publisher (Python)

import rclpy from std_msgs.msg import String rclpy.init() node = rclpy.create_node('publisher') pub = node.create_publisher( String, 'topic', 10 ) while rclpy.ok(): pub.publish(String(data="Hello")) rclpy.spin_once(node)
üîÑ Migration Path: While syntax differs, concepts remain similar. ROS2's rclpy closely mirrors rospy's API design, making migration relatively straightforward.

When to Use Each

Stick with ROS1 if:

  • Working with legacy code that's difficult to port
  • Need packages not yet ported to ROS2
  • Team expertise is entirely in ROS1
  • Non-critical applications (research, prototypes)

Choose ROS2 for:

  • New projects and production systems
  • Real-time drone control and person tracking
  • Multi-drone swarm applications
  • Commercial deployments requiring security
  • Embedded systems (Jetson Orin platforms)

1.1.3 DDS Middleware (FastDDS, CycloneDDS)

Data Distribution Service (DDS) is the communication backbone of ROS2. It's an industry-standard middleware that provides real-time, scalable, and reliable data exchange.

What is DDS?

üè≠

Industry Standard

OMG (Object Management Group) standard used in aerospace, defense, and industrial automation for decades.

üéØ

Publish-Subscribe

Data-centric architecture where publishers and subscribers communicate through topics without direct connections.

üåê

Peer-to-Peer Discovery

Automatic discovery of participants across networks without central broker or master node.

‚öôÔ∏è

Quality of Service

22+ QoS policies for fine-grained control over reliability, durability, latency, and resource usage.

DDS Architecture in ROS2

üì± Application: Your ROS2 Node

Drone control, computer vision, MAVROS communication

üîß RCL (ROS Client Library)

rclpy/rclcpp provides Node, Publisher, Subscriber, Service APIs

üîå RMW (ROS Middleware Interface)

Abstraction layer - allows switching between DDS implementations

‚ö° DDS Implementation

FastDDS, CycloneDDS, RTI Connext, etc.

üåê Network Transport

UDP multicast, UDP unicast, shared memory

Major DDS Implementations

F

Default in ROS2 Humble, Iron, Jazzy

Best for: Production systems, high throughput, discovery servers

Key DDS Concepts for Drones

üéØ Domain IDs

Isolate robot groups - each drone swarm can have its own domain (0-232). Default is 0.

export ROS_DOMAIN_ID=42

üì° QoS Policies

Configure reliability (RELIABLE vs BEST_EFFORT) for different sensor types - critical for camera streams.

üöÄ Shared Memory

Zero-copy transport for same-machine communication - huge benefit for YOLO inference on Jetson.

üîç Discovery

Automatic peer discovery via multicast. FastDDS Discovery Server reduces network chattiness.

üí° Recommendation for Your Drone Setup:
  • Jetson Orin Nano: Use CycloneDDS for lower memory/CPU usage
  • Multi-drone systems: Use FastDDS with Discovery Server
  • High-bandwidth cameras (SIYI): FastDDS with BEST_EFFORT QoS
  • Person tracking: Either works, CycloneDDS if CPU-constrained

Switching DDS Implementations

# Check current middleware echo $RMW_IMPLEMENTATION # Install CycloneDDS sudo apt install ros-humble-rmw-cyclonedds-cpp # Set in terminal session export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp # Or add to ~/.bashrc for permanent echo 'export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp' >> ~/.bashrc # Verify ros2 doctor --report | grep middleware
‚ö†Ô∏è Important: All nodes in your system must use the same DDS implementation. Mixing FastDDS and CycloneDDS nodes won't communicate!

Installation

# Humble on Ubuntu 22.04 sudo apt update && sudo apt install locales sudo locale-gen en_US en_US.UTF-8 sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8 export LANG=en_US.UTF-8 # Setup sources sudo apt install software-properties-common sudo add-apt-repository universe sudo apt update && sudo apt install curl -y sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \ -o /usr/share/keyrings/ros-archive-keyring.gpg echo "deb [arch=$(dpkg --print-architecture) \ signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \ http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \ | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null # Install ROS2 Humble sudo apt update sudo apt install ros-humble-desktop # Source setup echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc source ~/.bashrc # Verify installation ros2 --version

Which Distribution Should You Choose?

Choose Humble if:

  • Building production drone systems
  • Need proven stability (deployed worldwide)
  • Using Jetson Orin with JetPack 5.x
  • Want maximum package compatibility
  • Team is already familiar with it
  • Need commercial support options
üéØ Recommended for your person tracking drone project

Choose Jazzy if:

  • Starting a new long-term project
  • Want latest Python features (3.12)
  • Using Ubuntu 24.04 systems
  • Need cutting-edge Nav2 features
  • Can handle some package delays
  • Comfortable being early adopter
üöÄ Good for future-proofing new projects
üí° Migration Strategy: If starting now, go with Humble for stability. Plan to migrate to Jazzy around mid-2025 when the ecosystem matures. Both will be supported until 2027+.
